## 20. DIAGNOSTIC MQTT — PROCÉDURE COMPLÈTE (2026-04-10)

### Problème
Aucun message MQTT ne provient du Pi5 vers Node-RED depuis le dernier changement.

### Guides Disponibles

- **MQTT_RESTORE_IMMEDIATE.md** — Actions immédiates (⏱️ 2-3 min)
- **PI5_ACTION_PLAN.md** — Plan d'action avec 4 scénarios
- **MQTT_DEBUGGING_GUIDE.md** — Guide complet 8 étapes
- **QUICK_MQTT_DIAGNOSIS.sh** — Script de diagnostic automatisé

### Procédure Rapide (1 min)

**Sur le Pi5** :
```bash
cd ~/Daly-BMS-Rust
chmod +x QUICK_MQTT_DIAGNOSIS.sh
./QUICK_MQTT_DIAGNOSIS.sh
```

Cela affiche 5 vérifications essentielles :
1. État du service daly-bms (doit être `active (running)`)
2. Erreurs MQTT dans les logs
3. Configuration MQTT (host/port)
4. Connectivité broker (192.168.1.120:1883)
5. Messages MQTT reçus

### Action Corrective Rapide (30 sec)

Si aucun message MQTT :
```bash
# Copier la config à jour
sudo cp Config.toml /etc/daly-bms/config.toml

# Redémarrer le service
sudo systemctl restart daly-bms
sleep 2

# Vérifier
timeout 5 mosquitto_sub -h 192.168.1.120 -p 1883 -t 'santuario/bms/1/venus'
```

### Causes Usuelles

| Symptôme | Cause | Action |
|---|---|---|
| Service inactif | Code crashé ou erreur démarrage | `journalctl -u daly-bms -n 50` |
| Service actif, pas MQTT | Config stale (pas copiée sur /etc) | `sudo cp Config.toml /etc/daly-bms/config.toml` |
| Broker inaccessible | Mosquitto arrêté ou réseau | `make up` ou `ping 192.168.1.120` |
| Compilation échouée | Changement de code Rust | `make build-arm` + redéployer binaire |

### Workflow Complet en Cas de Changement Code/Config

```bash
# 1. Sur Claude Code/Pi5
git pull origin main

# 2. Compiler (si code modifié)
make build-arm

# 3. Redéployer
sudo systemctl stop daly-bms
sudo cp target/aarch64-unknown-linux-gnu/release/daly-bms-server /usr/local/bin/
sudo cp Config.toml /etc/daly-bms/config.toml
sudo systemctl start daly-bms

# 4. Vérifier
sleep 2
journalctl -u daly-bms -f
```

### Points Critiques à Retenir

1. **Config production** : Le service lit `/etc/daly-bms/config.toml`, PAS `~/Daly-BMS-Rust/Config.toml`
   → Toujours copier après modification

2. **MQTT config** : Vérifier `[mqtt]` section :
   ```toml
   [mqtt]
   enabled = true
   host = "192.168.1.120"    # IP NanoPi broker
   port = 1883
   topic_prefix = "santuario/bms"
   format = "json"
   ```

3. **Broker Mosquitto** : Tourne sur NanoPi (192.168.1.120:1883)
   → Vérifier avec `ping 192.168.1.120`

4. **Templates Askama** : Compilés dans le binaire
   → Les modifier seuls ne suffit pas, recompiler + redéployer

5. **Diagnostique systématique** : Toujours utiliser `QUICK_MQTT_DIAGNOSIS.sh` en premier
